# Park&Go

**Park&Go** is a web-based application that helps students, faculty, and visitors locate and compare affordable parking spots near the University of Minnesota Twin Cities campus. By providing search, filtering, and personalized suggestions, it aims to streamline the commuting experience and reduce the time spent looking for parking.

Static Page: [https://troppy2.github.io/Park-Go/](url)
## About

- **Problem:** Commuting to college can be frustrating, especially at UMN Twin Cities. Finding affordable and convenient parking is difficult and time-consuming.
- **Audience:** Students, faculty, and visitors traveling to the UMN Twin Cities campus.
- **Goal:** Provide personalized parking recommendations based on user preferences and input data.

## Features

- **Search bar:** Users can search for parking spots by name. Currently returns up to five similar results.
- **Filters:** Narrow down parking options for a more personalized experience.
- **Suggested Parking Spots:** Recommends parking locations based on user data and filters. *(Under development)*
- **Submit Parking Suggestions:** Users can submit new parking locations to expand the database.

## Problems / Future Development

- Develop the recommendation algorithm.  

**Current Problems:**  
- Codebase may be difficult to scale for larger projects.  
- CSS is repetitive and inefficient, leading to slow development and maintenance challenges.  
- HTML structure is suboptimal, making enhancements harder.

## Lessons Learned

- Using a modern front-end framework like **React** could reduce development time and improve maintainability.  
- Inefficient CSS can make seemingly simple tasks (like creating a sidebar) take hours.  
- Planning code structure early is essential for scalable web applications.
